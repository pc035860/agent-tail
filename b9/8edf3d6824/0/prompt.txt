### 4. **Drain Pattern in Handlers** (src/list/summary.ts, lines 68–87)

The drain mechanism shows how multi-part emission is actually consumed:

```typescript
/**
 * Parse lines through parser, collecting formatted output.
 * Handles stateful parsers (Claude) by draining with while loop.
 */
function parseAndFormat(
  lines: string[],
  parser: LineParser,
  formatter: Formatter
): string[] {
  const output: string[] = [];
  for (const line of lines) {
    let parsed: ParsedLine | null = parser.parse(line);
    let guard = 0;
    while (parsed &amp;&amp; guard &lt; 100) {
      const formatted = formatter.format(parsed);
      if (formatted) output.push(formatted);
      // Drain with empty string: stateful parsers (Claude) continue yielding
      // buffered parts; stateless parsers (Codex/Cursor/Gemini) return null
      parsed = parser.parse('');
      guard++;
    }
  }
  return output;
}
```

**Key details**:
- Line 75: `parser.parse(line)` is called **once** with the actual line
- Lines 77–84: **while loop** repeatedly calls `parser.parse('')` (empty string) to drain buffered parts
- Line 82 comment explicitly states: *"stateless parsers (Codex/Cursor/Gemini) return null immediately while Claude's `currentMessageState` continues draining"*
- **Guard clause** (line 76) prevents infinite loops at 100 iterations

**From CLAUDE.md (line 228)**:
&gt; "**`parseAndFormat` drain must use empty string**: In `summary.ts`, the while loop drains stateful parsers (Claude multi-part messages) by calling `parser.parse('')` — NOT `parser.parse(line)`. Stateless parsers (Codex/Cursor/Gemini) return the same result for the same line forever, causing 100x duplication. Empty string makes stateless parsers return null immediately while Claude's `currentMessageState` continues draining."

### 5. **contentToString Utility** (src/utils/text.ts, lines 70–102)

```typescript
/**
 * 將 content 轉為字串（處理物件/陣列情況）
 */
export function contentToString(content: unknown): string {
  if (typeof content === 'string') return content;
  if (content === null || content === undefined) return '';

  if (Array.isArray(content)) {
    return content
      .map((item) =&gt; {
        if (typeof item === 'string') return item;
        if (item &amp;&amp; typeof item === 'object') {
          // 處理 { type: 'text', text: '...' } 格式
          if ('text' in item &amp;&amp; typeof item.text === 'string') return item.text;
          // 處理 { type: 'tool_result', content: '...' } 格式
          if ('content' in item &amp;&amp; typeof item.content === 'string')
            return item.content;
          // 處理其他 type
          if ('type' in item) return `[${item.type}]`;
        }
        return '';
      })
      .filter(Boolean)
      .join(' ');
  }

  // 物件情況
  if (typeof content === 'object') {
    const obj = content as Record&lt;string, unknown&gt;;
    if ('text' in obj &amp;&amp; typeof obj.text === 'string') return obj.text;
    if ('content' in obj &amp;&amp; typeof obj.content === 'string') return obj.content;
    return JSON.stringify(content);
  }

  return String(content);
}
```

**Behavior with tool_use**: When an array contains `[{type: 'text', text: '...'}, {type: 'tool_use', name: '...', input: {...}}]`:
- The text object returns `.text` field
- The tool_use object **has no `.text` or `.content` property**, so it matches line 85: `return '[tool_use]'`
- Result: `"... text... [tool_use]"` (joined with space, line 90)

Used by **CursorLineParser** (line 14 import, line 450 usage) as a simple stateless flattener. Claude parser does NOT use this for `contentToString` — instead, it explicitly iterates each part in `parseAssistantMessage` and calls `processAssistantPart` for each one.

## Summary: Multi-Part Emit Mechanism

**For a single JSONL line containing `[text, tool_use, tool_use]`:**

1. **First `parse(line)` call** → `parseAssistantMessage(data, timestamp)` is invoked
   - Filters `content` array to keep only non-empty text and named tool_use
   - Initializes `this.currentMessageState = { contentParts: validParts, partIndex: 0, ... }`
   - Calls `processAssistantPart()` immediately
   - Returns **ParsedLine for the first text part** (type: `'assistant'`)

2. **Second `parse('')` call** (empty string from drain loop)
   - Checks `if (this.currentMessageState)` → true
   - Calls `processAssistantPart()`
   - Increments `partIndex` from 0 → 1
   - Returns **ParsedLine for the first tool_use** (type: `'function_call'`)

3. **Third `parse('')` call**
   - Checks `if (this.currentMessageState)` → true
   - Calls `processAssistantPart()`
   - Increments `partIndex` from 1 → 2
   - Returns **ParsedLine for the second tool_use** (type: `'function_call'`)

4. **Fourth `parse('')` call**
   - Checks `if (this.currentMessageState)` → true
   - Calls `processAssistantPart()`
   - `partIndex = 2 &gt;= contentParts.length (3)` → returns null (line 568)
   - Caller's while loop sees null, **clears `this.currentMessageState = null`** (line 457)

5. **Fifth `parse('')` call** (guard still active)
   - Checks `if (this.currentMessageState)` → **false**
   - Falls through to normal line parsing
   - `line = ''` is empty, returns null (line 461)
   - While loop exits (guard++, condition fails)

## Drain Contract for Implementers

**Stateful, multi-emit parser (like Claude):**
- Maintain **instance field** (e.g., `currentMessageState`) to queue parts
- On `parse(line)` with actual line: initialize state, return first ParsedLine
- On `parse('')` (empty string): process next buffered part, return ParsedLine or null
- When buffer exhausted, reset state to null
- **Contract**: Each call returns at most one item; state is mutated across calls

**Stateless, single-emit parser (like Cursor/Codex):**
- **No instance fields** that track parsing state
- On `parse(line)`: analyze only this line, return one ParsedLine or null
- On `parse('')`: return null immediately (empty line = no content)
- **Contract**: Always returns the same result for the same input
- **Consequence**: If consumer doesn't use the drain pattern, repeated calls yield repeated output (hence the guard + empty-string drain is essential)

---

The key design choice: **The interface returns `ParsedLine | null` (singular), not `ParsedLine[]`**. Multi-emission is achieved through **stateful instance fields** that survive across invocation boundaries, combined with a **caller-side while loop** that drains by passing empty strings until null is returned.</result>
