### `src/agents/codex/codex-agent.ts`

**[codex-agent.ts:75] 嚴重 — `findLatest` 現在對每個 JSONL 讀取第一行**

```ts
const meta = await readMainSessionMeta(file);  // 讀取整個檔案再 split('\n')[0]

```

`readMainSessionMeta` 在 `session-cache.ts:39` 用 `Bun.file(filePath).text()` 讀取**整個檔案**，再取第一行。Codex session 可能有數千行，但此處只需要第一行。當 Codex 有幾十個 session 時，`findLatest` 就要全量讀取所有 session 文件。

### `src/agents/claude/claude-agent.ts`

**[claude-agent.ts:43–92] 高 — `findLatest` 做全量 stat + sort 只為取第一個**

`_collectMainSessions` 對每個 JSONL 呼叫 `stat()`，建立完整 `SessionListItem[]`，全部排序，`findLatest` 取 `[0]`。Claude 的 `~/.claude/projects/` 可能有非常多歷史 session（多個 project、長期使用者數百個檔案）。

```ts
let latest: { path: string; mtime: Date; shortId: string } | null = null;
for await (const file of glob.scan(...)) {
  // filter logic...
  const stats = await stat(file);
  if (!latest || stats.mtime > latest.mtime) latest = { ... };
}
```

**[claude-agent.ts:52–88] 中 — glob 內的 stat 是序列化的**

建議：收集所有路徑後用 `Promise.all` 批次 stat，或使用原生 `Dirent` 搭配 `withFileTypes` 減少 syscall 次數。
