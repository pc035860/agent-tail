## Efficiency Review: `feat/nested-subagent-display`

### HIGH

**H1. `readSubagentMeta` worst-case latency = 300ms per registration** (`src/claude/subagent-detector.ts:123-146`)
Loop bound is `attempt &lt;= retries` (i.e. 6 iterations when `retries=5`), with a sleep on every non-final iteration. Worst case for a missing meta.json: 6 stat-checks + 5├Ś50ms sleeps = **250ms**, not 250ms inclusive. Then `lookupParentWithRetry` adds up to another **150ms** sequentially (4 attempts ├Ś 50ms ├Ś 3 sleeps). **H2. `scheduleSubagentsDirRetry` polls forever with no backoff** (`subagent-detector.ts:581-590`)
For a session that never spawns subagents (or where the user passed `--with-subagents` speculatively), this fires every 1000ms for the entire session lifetime. Each tick re-enters `tryWatchSubagentsDir` ŌåÆ catches ŌåÆ falls into `watchParentForSubagentsDir` ŌåÆ catches ŌåÆ reschedules. **Recommendation**: exponential backoff capped at e.g. 30s, or cap at N attempts (Claude Code typically creates the dir within the first few seconds if it's going to).
